# PR Metadata Checklist

Use this checklist before opening or reviewing a PR.

- [ ] Linked Linear issue is present.
- [ ] Role is present.
- [ ] Type is present.
- [ ] Risk is present.
- [ ] Validation profile is present.
- [ ] Summary is present.
- [ ] Files changed are listed.
- [ ] Tests run are listed.
- [ ] Manual QA is documented or marked not applicable with a reason.
- [ ] Conflict risk is documented.
- [ ] Visible UI expectation is documented.
- [ ] Known limitations are documented.
- [ ] Screenshots or video are included when visible UI or gameplay changed.
- [ ] PR targets `main`.
- [ ] PR is draft until validation and review are ready.
